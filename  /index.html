如何在Linux服务器上通过Hugo和Github搭建自己的博客

Posted by xuebin xu on Monday, January 1, 0001

在服务器上安装 Hugo

  1. 下载安装包：到Hugo官网，打开 Hugo 的 GitHub 源码，点击 Tags，选择最新版本的 Hugo 安装包，在 Assets 里选择适合自己服务器版本的安装包，我这里选择的是 hugo_extended_0.134.3_linux-amd64.tar.gz，右击复制链接地址，在服务器中输入：

    wget https://github.com/gohugoio/hugo/releases/download/v0.134.3/hugo_extended_0.134.3_linux-amd64.tar.gz

    下载 tar.gz 格式的安装包。

  2. 安装 hugo 并配置环境：进入存放 hugo 安装包的文件夹，将 hugo 安装包解压：

    tar -xzvf hugo_extended_0.134.3_linux-amd64.tar.gz -C [指定文件夹]

    安装完成后，打开服务上的 .bashrc 文件：

    vim ~/.bashrc

    将下面的代码复制到 .bashrc 文件中：

    export PATH=$PATH:/[...]/hugo_extended_0.134.3

    其中 /[...]/hugo_extended_0.134.3 是你存放 hugo 包的文件路径，替换成自己的就行。 然后输入：

    source ~./.bashrc

    输入下面的代码，检查 hugo 是否正确安装：

    hugo -version

    如果显示 hugo 的版本说明已经安装上了。

  3. 创建博客：

    hugo new site my-blog

    进入博客文件目录：

    cd my-blog

  4. 添加主题：在 hugo 官网主题库查找相应的主题，这里我以 Clean White 主题为例，按照该主题的说明文件，安装主题：

    cd themes
git clone https://github.com/zhaohuabing/hugo-theme-cleanwhite.git

    如果服务器无法通过 git 下载，可以直接使用 url 下载后，解压：

    cd themes
wget https://github.com/zhaohuabing/hugo-theme-cleanwhite/archive/refs/heads/master.zip
unzip master.zip

    下载主题完成后将主题中的 exampleSite 中的文件拷贝出来到自己博客文件夹：

    cp -r hugo-theme-cleanwhite/exampleSite/** ../
cd ..

    此时，可以在本地运行博客，打开网址查看：

    hugo server -D

  5. 更改模板：此时的博客还是模板形式，需要对里面的内容进行修改：

    • /content/about/: 此文件夹是放个人简介文件位置；
    • /content/notes/: 此文件夹是放笔记的位置；
    • /content/post/: 此文件夹是放博客内容的位置，生成的博客都在此处；
    • config.toml 文件：此文件是博客模板格式的文件，可根据相关内容进行修改。

  6. 创建新的博客：

    hugo new posts/my-first-post.md

    此时创建了新的博客 .md 文件，可打开添加博客内容。

  7. 使用 GitHub 托管博客：首先你得有一个 GitHub 账号： 登录账号后，创建博客仓库，Repository name 设置为 [username].github.io，[username]是自己的 GitHub 账户名，勾选 Public 和 README 文件，设置 main 分支为仓库默认分支。 GitHub 账号注册和仓库创建详见教程。

  8. 将博客推送至 GitHub 仓库：首先生成静态文件，静态文件存放在 public 文件夹中：

    hugo

    进入 public 文件夹，初始化 Git，链接远程 GitHub 服务器：

    cd public
git init
git remote add origin https://github.com/<username>/<username>.github.io.git

# 提交推送
git add .
git commit -m "Initial commit"
git push -u origin main

    可能出现的错误： （1） error: src refspec main does not match any. error: failed to push some refs to 查看 branch 名称：

    git branch

    如果分支名称是 mster，则需要将其修改与 GitHub 上一致：

    git branch -m master main

    （2）fatal: unable to access: Failed to connect to github.com port 443: Connection timed out 查看网络连接是否正常：

    ping github.com

    如果正常，尝试用 set-url 链接

    git remote set-url origin github@github.com:<username>/<username>.github.io.git
ssh -T git@github.com

    如果显示下面结果说明连接成功：

    Hi XXX! You've successfully authenticated, but GitHub does not provide shell access.

    此时再重新推送。 （3）如果出现 ! [rejected] main -> main (fetch first) error: failed to push some refs to 可先将 GitHub 上的仓库先 pull 下来，然后再推送：

    git pull origin main

    如果还是不行，可以强制推送（此方法不建议）：

    git push origin main --force
